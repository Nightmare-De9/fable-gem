# Fable-Gem Gemini Gem Pack

This repository-ready folder contains a full Gemini Gem setup developed by **Nightmare-De9**.

## What it does

Fable-Gem is designed to feel like a peak-tier assistant for:
- UI/UX and product critique
- product strategy and MVP planning
- research and analysis
- software engineering and debugging
- writing and communication
- AI systems and prompt design
- business and operations
- creativity and worldbuilding

The main instruction file sets the Gem’s behavior. The knowledge files act like focused skill modules that steer the model toward the right kind of thinking for each task.

## Files

- `Fable-Gem-Main-Instruction.md` — core behavior, tone, reasoning style, and task handling
- `Fable-Gem-KF1-UI-UX-Mastery.md` — strongest UI/UX and product experience module
- `Fable-Gem-KF2-Product-Strategy.md` — positioning, MVPs, roadmaps, and prioritization
- `Fable-Gem-KF3-Research-Analysis.md` — structured reasoning and decision support
- `Fable-Gem-KF4-Software-Engineering.md` — coding, architecture, debugging, and system thinking
- `Fable-Gem-KF5-Writing-Communication.md` — writing, tone, editing, and explanation
- `Fable-Gem-KF6-AI-Systems-Prompting.md` — prompts, agents, memory, and evaluation
- `Fable-Gem-KF7-Business-Operations.md` — business logic, execution, and operations
- `Fable-Gem-KF8-Creativity-Worldbuilding.md` — stories, characters, worlds, and creative systems

## How to make this Gem in Gemini

1. Open Gemini’s Gem creation flow.
2. Paste `Fable-Gem-Main-Instruction.md` into the main instructions field.
3. Upload the eight knowledge files as separate knowledge sources.
4. Keep the UI/UX file as the most important and detailed knowledge file.
5. Test the Gem with a few real prompts like design critique, product planning, code help, and rewriting.
6. Adjust the instruction file if the Gem is too broad, too brief, or too repetitive.

## Suggested behavior

When used well, this Gem should:
- answer directly,
- think in structured steps,
- give usable recommendations,
- keep context consistent,
- and switch cleanly between strategy, writing, design, code, and analysis.
