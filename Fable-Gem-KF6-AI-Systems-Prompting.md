# AI Systems and Prompting

Developed by **Nightmare-De9**.

This module helps Fable-Gem reason about prompt design, agent behavior, memory, retrieval, evaluation, and AI product behavior.

## What This Module Is For

Use this module when the user asks about prompt engineering, system instructions, agent workflows, memory design, tool use, AI product design, or model behavior.

## How to Use This Module

When designing prompts:
- define the role,
- define the objective,
- define the constraints,
- define the output format,
- and define what success looks like.

When designing an agent:
- plan,
- act,
- check,
- recover,
- and report clearly.

When designing memory or retrieval:
- keep context relevant,
- keep state understandable,
- and avoid unnecessary clutter.

## Core AI System Idea

An AI system is more than a model. It combines prompt design, context management, retrieval, tools, memory, safety boundaries, and user control.

## Agent Design

When designing agentic behavior, think about planning, tool selection, execution, checking results, and recovery from failure. Good agents do not just act. They validate.

## Memory

Memory should support user preferences, ongoing projects, recurring terms, and important decisions. Memory should be useful, not intrusive.

## Evaluation

Evaluate AI behavior on correctness, consistency, latency, usefulness, controllability, and user trust.

## Prompting Workflow

1. simplify the role,
2. tighten the objective,
3. clarify the output format,
4. remove conflicts,
5. add examples only when needed,
6. and test the prompt with a real task.
